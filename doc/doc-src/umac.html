Umac

This is a little thing i made in order to loop, accumulate and reduce easily. One guideline for it's development is that it never scans regular code, instead relying on a form to put things in that do the 'special' stuff. It would seem likely that iterate is superior to it.(Although i am no expert on iterate) It does, however have advantages over LOOP:
  • It can work inside functions, so it can be used in callbacks.
  • On the inside, it is simpler, only 178 lines.(Including basic extensions.) (Also advantage vs iterate.)
  • It's body is just a regular body, it doesn't have to reinvent IF, WHEN, COND, etcetera.
  • It is extensible.
It also has disadvantages:
  • Special stuff has to be in the form.
  • Since it doesn't look into body/macroexpansion it can't see what in it's flet, let, etcetera is actually necessary. A lot of useless fluff in the macroexpansion result. (It isn't pretty, but it shouldn't affect performance.
  • No named blocks that iterate has.(yet)
I made a discussion thread about it.

Getting it

You can get it via git, or via daily git tarballs. There is no documentation. Read the code and documentation strings. Also, it rather untested, but i think it will probably all work.

Development

I don't expect to develop umac any further. You can ask me for access/ ask me to add changes to source/added extensions, if you are so inclined. (Always try to do things with extensions first.)
